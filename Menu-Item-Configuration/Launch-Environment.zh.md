# 工作目录

设置启动进程的初始工作目录。

**默认值：** `[dir]` — 右键点击文件的父文件夹。

## 可用变量

| 变量 | 说明 | 示例值 |
|------|------|--------|
| `[folder]` | 文件夹本身（文件夹）或父文件夹（文件） | `C:\Docs` |
| `[dir]` | 父文件夹路径 | `C:\Docs` |
| `[dirname]` | 父文件夹名称 | `Docs` |
| `[stem]` | 不带扩展名的文件名 | `report` |
| `[name]` | 带扩展名的文件名 | `report.pdf` |

> **提示：** 如果你的菜单项同时针对文件和文件夹，请使用 `[folder]` 代替 `[dir]`——它始终解析为相关文件夹。详见 [启动参数 → `[folder]` 与 `[dir]` 的区别](Launch-Arguments.zh.md#folder-与-dir-的区别)。

> **注意：** 此处不支持带引号的变体（`[dir:q]`、`[path:q]`、`[folder:q]`）和文件扩展名变量（`[ext]`），因为工作目录必须是原始、不带引号的路径——`ShellExecute` 的 `lpDirectory` 参数不接受带引号的路径。

> **注意：** 当工作目录字段留空时，默认为右键点击文件的父文件夹（`[dir]`）。

## JSON 配置

```json
{
    "WorkingDirectory": "[dir]"
}
```

接受任何包含路径变量的字符串。如果省略或为空，默认为右键点击文件的父文件夹。
